# /commit

このプロジェクトへの git コミットを行います。

## 手順

1. `git status` で未追跡ファイルと変更を確認する
2. `git diff` でステージ済み・未ステージの変更を確認する
3. `git log --oneline -5` で直近のコミットメッセージスタイルを確認する
4. 変更内容を分析してコミットメッセージを作成する:
   - 日本語でメッセージを書く（このプロジェクトのコミット履歴に合わせる）
   - 変更の「何を」より「なぜ」を重視した簡潔なメッセージにする
   - 以下のprefixを変更内容に応じて先頭に付ける:
     - `feat:` 新しい機能
     - `fix:` バグの修正
     - `docs:` ドキュメントのみの変更
     - `style:` 空白、フォーマット、セミコロン追加など
     - `refactor:` 仕様に影響がないコード改善(リファクタ)
     - `perf:` パフォーマンス向上関連
     - `test:` テスト関連
     - `chore:` ビルド、補助ツール、ライブラリ関連
5. ユーザーが承認した場合のみ、関連ファイルをステージングエリアに追加してコミットを作成する

## 注意事項

- ユーザーに明示的に依頼された場合のみコミットする
- `git push` はユーザーに明示的に依頼された場合のみ行う
- 機密情報を含むファイル（`.env`、認証情報など）はコミットしない
- コミットメッセージは以下の形式で作成する:

```
<prefix>: <変更の要約>

<変更の理由や詳細な説明（必要に応じて）>
```
