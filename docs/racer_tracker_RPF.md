# ボートレーサーデータ収集プログラム要求依頼書

## 目的
公式サイトからボートレーサーの選手情報と成績情報データを収集しデータ処理しやすい形で保存するプログラムを作成すること。

## 現状課題
- ボートレーサーのデータは公式サイトから手動でダウンロードする必要があり、効率が悪い。
- データの形式がテキストの固定バイト長で提供されているため、分析のためにデータを整形する必要がある。
- 公式サイトのデータはlzh形式で提供されているため、解凍する手間がかかる。
- データの収集と整形・保存のプロセスが自動化されていないため、定期的なデータ更新が困難である。

## 要求事項
- 年と前期・後期を指定してボートレーサーのデータを公式サイトからダウンロードすること。
- ダウンロードしたデータを解凍し、テキストファイルから選手情報と成績情報を抽出すること。
- 収集したデータ分析しやすい形で保存すること。
- プログラムはコマンドラインから実行できること。
- データの保存形式はCSVやJSONなどの一般的な形式であること。
- データの収集と保存のプロセスは自動化されていること。
- データの収集と保存のプロセスはエラー処理が適切に行われること。

## 非機能要求
- プログラムはBashのシェルスクリプトとPythonで実装されること。
- プログラムはLinux環境で動作すること。
- プログラムは定期的に実行されることができるようにcronジョブなどでスケジュールされることを想定していること。
- プログラムはエラーが発生した場合に適切なログを出力し、エラーの内容を明確にすること。

## データソース
- ボートレーサーのデータは、公式のボートレースサイトや関連するAPIから収集されることが想定されていること。
- `https://www.boatrace.jp/owpc/pc/extra/data/download.html` のレーサー期別成績ダウンロードから収集すること
